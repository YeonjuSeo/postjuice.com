## Vite 개발 서버 캐시 이슈(504 Outdated Optimize Dep)

```
GET http://localhost:5173/node_modules/.vite/deps/yaml.js?v=… net::ERR_ABORTED 504 (Outdated Optimize Dep)
```

- 프리번들 캐시가 바뀌었는데도 브라우저는 예전 `…/deps/무언가.js?v={해시}`로 요청하고, Vite 개발 서버는 그 버전을 더 이상 유효하지 않다고 보아 504로 끊겨 React가 로드되지 않음

### 문제 발생 흐름

| 단계 | 주체               | 일어나는 일                                                                                                                                                                         |
| ---- | ------------------ | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 1    | **나**             | 터미널에서 `npm run dev` 를 실행한다.                                                                                                                                               |
| 2    | **내 PC**          | Vite 개발 서버(Node **프로세스**)가 뜬다.                                                                                                                                           |
| 3    | **Vite 프로세스**  | `node_modules` 를 보며 프리번들할 대상 목록 생성                                                                                                                                    |
| 4    | **디스크**         | `node_modules/.vite/deps/` 에 합쳐진 JS 를 쓰고, 브라우저가 부를 **URL(파일명?v=해시)** 을 준비한다.                                                                                |
| 5    | **나**             | 크롬에서 `http://localhost:5173` 을 연다.                                                                                                                                           |
| 6    | **브라우저**       | `index.html` → `main.tsx` → … 를 거쳐 `…/deps/yaml-….js?v={해시_A}` 같은 요청을 보낸다.                                                                                             |
| 7    | **Vite 개발 서버** | 해당 파일을 **정상적으로** 응답한다. (**요청의 `v` 와 서버가 들고 있는 스냅샷이 일치**)                                                                                             |
| 8    | **브라우저**       | JS 가 실행되고 React 가 마운트되어 **화면 렌더링**                                                                                                                                  |
| 9    | **나**             | 서버를 끄지 않은 채 `package.json` 의존성을 바꾸고 `npm install` 한다. 디스크의 **실제 패키지**는 이미 새 상태임.                                                                   |
| 10   | **Vite 프로세스**  | **예전부터 켜 둔 채**이면 메모리 안의 “유효한 deps 스냅샷”이 옛날이거나, 갱신이 끝나기 전 **레이스**로 서버가 인정하는 `?v=` 집합이 바뀌었는데 브라우저가 아는 것과 어긋날 수 있다. |
| 11   | **디스크 `.vite`** | 새 의존성과 캐시 내용이 잠깐 안 맞거나, 옛 청크·옛 해시가 남아 있을 수 있다.                                                                                                        |
| 12   | **브라우저**       | 강력 새로고침을 안 했거나, **예전에 받은** 스크립트·탭 상태가 `yaml.js?v=**옛 해시**` 같은 URL 로 다시 요청한다.                                                                    |
| 13   | **Vite 개발 서버** | “지금은 그 `v` 가 유효하지 않다”고 보고 **504 Outdated Optimize Dep** 쪽으로 요청을 끊는다.                                                                                         |
| 14   | **브라우저**       | `yaml` 스크립트가 로드되지 않아 **React 가 안 뜨고** `#root` 가 빈 화면으로 남는다.                                                                                                 |

### Solution

1. `npm run dev` 다시(Vite 재시작) ➡️ 프로세스 메모리
2. `node_modules/.vite` 삭제 ➡️ 디스크 캐시
3. 브라우저 강력 새로고침 ➡️ 브라우저 캐시

셋 중 하나 하면 해결...

\+ `vite.config.ts`의 `optimizeDeps.include: ['yaml']`

- 원소를 항상 의존성 사전 번들(deps 최적화) 후보에 넣어서 최적화 목록이 달라짐 -> 해시 값이 달라지는 경우 완화
